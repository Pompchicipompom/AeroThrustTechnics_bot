# Секретные данные

Документ описывает секретные данные, необходимые для промышленного запуска.  
Реальные значения в репозиторий и в настоящий файл не записываются.

## Правила хранения

- Production `.env` должен храниться только на сервере
- Реальные секреты не должны храниться в git, общих архивах, открытой переписке и скриншотах
- Рекомендуется заполнять `.env` непосредственно на сервере из шаблона `.env.production.example`
- Доступ к секретам следует ограничивать кругом администраторов системы

## Обязательные секретные данные

| Данные | Переменная / способ задания | Назначение |
|--------|-----------------------------|------------|
| Токен Telegram-бота | `TELEGRAM_BOT_TOKEN` | Работа контейнера `bot` |
| Пароль PostgreSQL | `POSTGRES_PASSWORD` | Инициализация БД |
| Строка подключения к БД | `DATABASE_URL` | Доступ backend и bot к PostgreSQL |
| Секрет подписи JWT | `ADMIN_JWT_SECRET` | Авторизация admin API |
| Логин администратора | CLI `create-admin-user --email` | Вход в админку |
| Пароль администратора | CLI `create-admin-user --password` | Вход в админку |

Формат `DATABASE_URL`:

```text
postgresql+asyncpg://USER:PASSWORD@postgres:5432/DB
```

Пароль в `DATABASE_URL` должен совпадать с `POSTGRES_PASSWORD`.

Для промышленной среды также требуется `APP_ENV=prod`.

## Дополнительно

| Данные | Переменная / примечание |
|--------|-------------------------|
| Публичный порт админки | `ADMIN_HOST_PORT` |
| Invite-коды | создаются в таблице `invite_codes`, не через env |
| Учётные записи resolver | опционально через `create-admin-user --role resolver --zone ...` |
| Домен / URL админки | используются при настройке TLS reverse proxy |

## Опциональные параметры безопасности и лимитов

| Параметр | Переменная |
|----------|------------|
| Принудительный IPv4 к Telegram | `BOT_FORCE_IPV4` |
| Rate limit бота | `BOT_RATE_LIMIT_*` |
| Лимиты вложений и текста | `MAX_ATTACHMENT_*`, `MAX_REPORT_TEXT_LENGTH` |
| Разрешённые типы файлов | `ALLOWED_DOCUMENT_*` |

## Чеклист перед запуском

- [ ] В репозитории отсутствует файл `.env` с реальными значениями
- [ ] В `.env.production.example` присутствуют только плейсхолдеры
- [ ] `ADMIN_JWT_SECRET` уникален и достаточно длинный
- [ ] Пароли администратора и БД не совпадают с примерами из документации
- [ ] При компрометации токена Telegram выполнена его ротация
