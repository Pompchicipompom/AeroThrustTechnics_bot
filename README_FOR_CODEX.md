# Aerotrust Bot — стартовый пакет для Codex

В папке лежат документы, с которых стоит начать разработку MVP Telegram-бота для системы доверенных сообщений.

## Что внутри
- `01_product_brief.md` — краткая продуктовая концепция
- `02_tech_spec_mvp.md` — функциональное ТЗ на MVP
- `03_architecture_and_stack.md` — рекомендуемая архитектура и стек
- `04_env_example.env` — пример переменных окружения
- `05_codex_first_prompt.md` — первый промпт для Codex
- `06_project_structure.md` — рекомендуемая структура репозитория

## Как использовать
1. Положить файлы в корень репозитория или в папку `/docs`.
2. Открыть репозиторий в Codex.
3. Сначала дать Codex файл `05_codex_first_prompt.md`.
4. После генерации каркаса проекта идти итерациями: бот → БД → админка → аналитика → деплой.

## Важно
Не хранить реальные секреты в репозитории:
- Telegram bot token
- SSH-пароли
- пароли БД
- ключи API

Использовать только `.env` и секреты окружения.
