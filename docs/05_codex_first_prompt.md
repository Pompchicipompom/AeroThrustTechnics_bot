# Первый промпт для Codex

Ты работаешь в репозитории проекта MVP системы доверенных сообщений Aerotrust Technics.

## Контекст
Нужно реализовать MVP системы с двумя интерфейсами:
1. Telegram-бот для отправки сообщений
2. Административная веб-панель для обработки сообщений

Основные документы проекта:
- `docs/01_product_brief.md`
- `docs/02_tech_spec_mvp.md`
- `docs/03_architecture_and_stack.md`
- `docs/06_project_structure.md`

Перед началом внимательно прочитай эти документы и используй их как source of truth.

## Цель этой первой итерации
Сейчас не нужно пытаться сделать весь продукт сразу. Нужно подготовить качественный фундамент проекта.

## Что нужно сделать на первой итерации
1. Создать каркас проекта по структуре из `docs/06_project_structure.md`.
2. Настроить backend на Python 3.12 с FastAPI.
3. Подключить aiogram 3 для Telegram-бота.
4. Настроить PostgreSQL и SQLAlchemy 2.
5. Подключить Alembic.
6. Подготовить Docker Compose для:
   - backend
   - postgres
   - redis
7. Создать базовые ORM-модели:
   - User
   - InviteCode
   - Report
   - Attachment
   - AdminUser
   - AuditLog
8. Подготовить первичную миграцию.
9. Реализовать базовый healthcheck endpoint.
10. Реализовать минимальный bot bootstrap:
    - команда `/start`
    - проверка авторизации пользователя
    - запрос invite code, если пользователь не авторизован
11. Реализовать базовую логику invite code:
    - проверка кода в БД
    - отметка пользователя как authorized
12. Подготовить README по локальному запуску.
13. Подготовить `.env.example`.
14. Если чего-то не хватает, предложить разумные технические решения и зафиксировать их в README.

## Важные требования
- Не использовать монолитный файл на тысячу строк.
- Код должен быть модульным и чистым.
- Использовать pydantic settings / env-based config.
- Добавить type hints.
- Добавить базовую обработку ошибок.
- Не хардкодить секреты.
- Не делать фейковую реализацию — нужен рабочий foundation.
- Если admin UI пока не реализуешь, подготовь для него пустую структуру и README.

## Что нужно вернуть в результате
1. Изменения в коде.
2. Краткий отчет:
   - что сделано;
   - что не сделано;
   - какие следующие шаги логично делать.
3. Инструкцию запуска.

Начинай с анализа repo, затем создай foundation проекта и реализуй первую итерацию без лишнего scope creep.
