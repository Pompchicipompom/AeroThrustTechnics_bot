# Product Brief — Aerotrust Trusted Messages Bot

## 1. Название
**Aerotrust Trusted Messages Bot**  
Русское рабочее название: **Система доверенных сообщений Aerotrust Technics**

## 2. Идея продукта
Сервис на базе Telegram-бота и административной веб-панели для приема добровольных сообщений о:
- рисках;
- нарушениях;
- неприемлемых событиях;
- проблемах в процессах;
- предложениях по улучшению.

Пользователь может отправить сообщение:
- **анонимно**
- **открыто**

## 3. Ценность
Система снижает барьер для сообщения о проблемах и дает компании дополнительный канал раннего выявления рисков и слабых мест.

## 4. Целевая аудитория
Пользоваться ботом может любой человек, получивший доступ:
- через ссылку на бота;
- через invite code.

Это могут быть сотрудники, подрядчики или иные лица с выданным доступом.

## 5. Главные принципы
1. Простота подачи сообщения.
2. Доверие к анонимному формату.
3. Маршрутизация по зонам ответственности.
4. Минимально достаточная аналитика для управления.
5. Безопасное хранение данных.

## 6. Формат MVP
### Пользовательская часть
- вход по invite code;
- выбор режима: анонимно / открыто;
- выбор категории;
- отправка текста;
- возможность приложить фото / файл;
- подтверждение: «Сообщение принято».

### Внутренняя часть
- веб-панель;
- список сообщений;
- фильтры по зонам и статусам;
- просмотр карточки сообщения;
- смена статуса;
- аналитика.

## 7. Категории сообщений (MVP)
- Безопасность
- Качество
- Процессы
- Поведение / этика
- Улучшения
- Другое

## 8. Статусы (MVP)
Внутренние:
- New
- In Progress
- Closed

Для отправителя:
- только подтверждение успешной отправки.

## 9. Ключевое обещание анонимности
При выборе анонимного режима **получатель сообщения не должен видеть данные отправителя**.

Важно: формулировать честно. Не обещать абсолютную техническую неотслеживаемость без соответствующей архитектуры и аудита.
